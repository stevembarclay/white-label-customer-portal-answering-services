# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public GitHub issue**. Public disclosure before a fix is available puts other users at risk.

Instead, report it privately:

- **Email:** steve.m.barclay@gmail.com
- **Subject line:** `[SECURITY] Brief description`

Include in your report:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

You'll receive an acknowledgment within 48 hours. We aim to release a fix or mitigation within 14 days for critical issues.

## Scope

This policy covers the code in this repository. It does not cover:

- Third-party services (Supabase, OpenAI, Vercel) — report those to their respective teams
- Issues in dependencies — report to the upstream package maintainer

## Supported Versions

Security fixes are applied to the latest release only. We do not backport fixes to older versions.

## Disclosure Policy

Once a fix is released, we'll publish a security advisory on GitHub. We credit reporters by name (or anonymously, if preferred).
